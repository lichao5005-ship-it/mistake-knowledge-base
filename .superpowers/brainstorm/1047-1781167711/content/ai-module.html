AI 模块设计 — 识别 → 判错 → 分析 → 出题

全部由大模型视觉能力驱动，统一 AI 适配层

架构策略：端到端大模型方案

既然你选了 AI 方案，建议直接用 多模态大模型（GPT-4o / Claude Sonnet） 一次性完成"看到→理解→判断→分析"全流程，而非拆成传统 OCR → NLP → 规则的多阶段流水线。

AI 模块内部结构

modules/ai/
├── ai_client.py           # AI 统一适配层（支持切换模型）
│   ├── call_ai(model, prompt, image) → 通用接口
│   └── retry / fallback / token 统计
│
├── ocr_service.py        # 图片 → 结构化题目数据
│   ├── extract_questions(image) → [{number, content, blank}]
│   └── 处理：语文识字 / 数学公式 / 英语单词
│
├── correction_service.py # 判错引擎
│   ├── check_answer(question, student_answer) → {correct, reason}
│   └── 支持：填空题/选择题/计算题/应用题
│
├── classify_service.py   # 自动分类
│   ├── classify_subject(text) → 语文/数学/英语
│   ├── classify_knowledge_point(text) → 知识点ID
│   └── classify_error_type(question, answer) → 错误类型
│
├── generate_service.py   # 出题引擎
│   ├── generate_similar(question) → [similar_questions]
│   └── generate_paper(subject, kps, count) → exam_paper
│
└── analysis_service.py   # 分析引擎
    ├── analyze_weakness(mistakes) → weakness_report
    └── generate_report(mistakes, period) → study_report

一次完整的录入 = 3 步 AI 调用

1 OCR 识别

发送裁剪后的图片给 AI

返回 JSON：每道题的题号、题目内容、空白位置

"识别这张图片中的所有题目，按题号返回 JSON 格式，包含题目内容和空白/作答区域"

2 判错 + 分类

对每道题，AI 判断对错

标注错误类型，关联知识点

"题目内容：32×15，学生答：460。判断对错，分析错误原因，标注知识点"

3 正解分析

AI 给出正确答案和解题思路

特为小学生设计：用孩子能懂的语言解释

"用一年级学生能理解的方式，写出正确答案和解题步骤"

三科特殊处理

科目 识别难点 应对策略
语文 手写生字、阅读理解长文本、作文 直接图片→AI，注重大量手写汉字识别精度
数学 分数/除法竖式/几何图形/单位换算 AI 理解图形语义，支持 LaTeX 格式输出公式
英语 手写字母、选择题 ABCD 相对简单，AI 直接处理

出题 Prompt 策略

相似题生成

基于原题知识点+难度，出 3 道同类题

原题是"32×15"，出 3 道两位数乘法的计算题，附答案和解题步骤

模拟卷生成

选择知识点和难易度，AI 组卷

数学三年级上册期中模拟卷，包含：填空 5 题、计算 4 题、应用 3 题，覆盖四则运算和长度单位

关键设计原则

  • 结构化输出 — AI 一律返回 JSON，前端直接渲染
  • 统一 AI 适配层 — 所有模块调 ai_client.call()，换模型只改一处
  • 结果可修正 — AI 识别结果允许家长在 PC 端手动编辑
  • Prompt 模板管理 — 每个场景的 prompt 存独立文件，方便调优
