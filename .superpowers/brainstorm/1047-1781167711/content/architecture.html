推荐架构方案对比

基于你的需求：模块独立、数据库解耦、易于扩展

方案 A：Python 模块化单体（推荐）

后端：FastAPI 前端：Vue 3 数据库：SQLite → PostgreSQL ORM：SQLAlchemy
模块结构：

优势

  • 模块高度独立 — 每个模块有自己的一套 router + service + models，修改一个模块不影响其他
  • 数据库按模块拆分 — 每个模块的 models.py 只定义自己的表，共享数据放 shared/
  • 未来可拆微服务 — 把某个 module 整个拎出去变成独立服务，改几行路由前缀即可
  • Python AI 生态 — 调用 Claude/GPT API、PDF 解析、图片处理最方便
  • SQLite 起步 — 本地开发零配置，需要时可以无缝切到 PostgreSQL

注意点

  • Vue 前端需要一点前端开发量
  • 相比全栈 JS，前后端之间多一层 API 规范

方案 B：全栈 TypeScript（NestJS + Next.js）

后端：NestJS 前端：Next.js 语言：TypeScript 统一 ORM：Prisma

优势

  • 前后端同语言，类型共享
  • NestJS 模块系统本来就是模块化设计
  • Next.js 前端 SSR，体验好

注意点

  • 调 AI API 需要额外封装（Python 生态更直接）
  • OCR/PDF 处理在 JS 生态中相对薄弱
  • 对不熟悉前端的人来说配置项较多

我的推荐：方案 A — Python 模块化架构

理由：你的核心业务（OCR、AI 判错、AI 出题、分析）全部依赖 AI/LLM 能力，Python 是这方面生态最成熟的语言，开发效率最高。FastAPI 原生支持模块化拆分，数据库按模块独立设计，后续扩展非常清晰。

前端用 Vue 3 做一个独立的 SPA，通过 API 和后端通信，将来前端也可以拆成独立项目。
